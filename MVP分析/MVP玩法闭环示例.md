### 1. 重新梳理：一个更聚焦的MVP玩法闭环

你的闭环整体合理，但有些部分可以更极致地“最小化”，确保核心体验先跑通。我帮你把它梳理成清晰的**用户动作**和**系统动作**对照表，并做了轻微简化，把“多agent团队”先放到拓展区，让MVP回归最纯粹的“单人+AI主持人”体验。

**核心闭环流程表：**

| 阶段 | 用户做什么 | 系统做什么 |
| :--- | :--- | :--- |
| **1. 开局** | 从3-5个预生成的世界/剧本中选择一个（例如：1920年代神秘庄园调查） | 展示世界观简介、核心任务。后台已为该世界预设了完整的系统提示词和初始世界状态。 |
| **2. 创角** | 查看预生成的角色卡，可一键选用，或简单修改名字、调整属性分配。 | 根据所选世界，生成3-5张符合风格的角色卡（含属性、技能、简短背景）。主角物品栏初始化为空。 |
| **3. 开幕** | 点击“开始故事”，阅读AI生成的开场叙事。 | **严格按“Meta-Narrative”格式输出**：首段旁白介绍场景、主角、开幕事件。**不输出任何选项**，纯粹让用户沉浸。 |
| **4. 首次交互** | 输入行动（如“我搜索书房的桌子”），点击发送。 | 1. **后台判定**：识别这是行动，触发“Normal Narrative”模式。内部思考流程启动，结合世界观、角色卡、当前场景、潜在骰子规则（先不告知）。<br>2. **输出叙事+选项**：输出剧情推进。**结尾固定提供3个参考行动选项 + 1个“自由行动”输入框**。 |
| **5. 关键物品获取** | 按提示发现“一把带血的钥匙”，叙事中提及。 | **后台自动记录**：将“带血的钥匙”加入主角物品栏，并在UI侧边栏独立显示。 |
| **6. 元叙事干预（可选）** | 点击“暂停故事/Meta模式”按钮，输入：“我觉得刚才的钥匙应该藏得更隐蔽。” | 切换至Meta模式，暂停剧情推进。理解用户的叙事修改意图，调整后台对“钥匙”的存储描述，并生成一段简短说明“已记录”。玩家可退出Meta模式继续。 |
| **7. 循环与中断** | 继续输入行动，或随时关闭页面。 | 每次用户输入后自动保存。下一次用户打开时，从上次中断处继续，并回顾前一段剧情。 |
| **8. 结局** | 触发关键剧情，达成结局。 | 系统判定故事到达预设终点，输出结局叙事。生成“本次冒险总结”，并更新玩家角色卡的背景故事（可选）。 |

这个闭环的改进点在于：
-   **强制开场无选项**：避免初次接触的混乱，强化代入感。
-   **Meta模式独立入口**：把“修改剧情”和“推进剧情”两个动作彻底分开，降低AI逻辑负担。
-   **物品系统自动化**：让物品管理成为系统后台逻辑，不给AI增加记忆压力，也能保证严谨。

### 2. 盘点实现：技术与程序/UI搭建

这个盘点会偏重“你需要什么”，不写代码，但会给出技术选型方向，方便你用Vibe coding工具去实现。

#### 🧠 技术架构清单

| 模块 | 核心功能 | 实现方案/技术选型 |
| :--- | :--- | :--- |
| **大语言模型(LLM)** | 所有叙事、思考、判定的核心引擎。 | 使用API调用：OpenAI GPT-4o/Claude 3.5 Sonnet，或性价比更高的国产大模型（DeepSeek、Kimi等）。初期用一个模型就够了。 |
| **系统提示词工程** | 定义AI为“KP/DM”，规定其思考流程、输出格式、禁用行为。 | 为每个世界观编写独立的、结构化的系统提示词（System Prompt），明确角色、规则、输出范式（Meta/普通）。这是产品的灵魂。 |
| **记忆与状态管理** | **短期**：记住最近的对话。<br>**长期**：记住角色信息、物品、关键剧情。 | **短期**：使用滑动窗口，保留最近N轮对话。<br>**长期MVP方案**：用**摘要压缩**。每过10轮对话，让AI自动将历史压缩成300字摘要，存入数据库。**暂时不用RAG**，降低复杂度。 |
| **物品/角色卡系统** | 独立于AI之外，精确记录物品和属性。 | **数据库**：使用轻量级数据库（如Supabase/PostgreSQL）存储角色ID、物品列表、属性值等结构化数据。<br>**逻辑**：AI叙事时，用“函数调用”要求它输出`{ "add_item": "带血的钥匙", "desc": "..." }`，后端解析后更新数据库。UI从数据库读取展示。 |
| **骰子与外置规则** | 保证公平和随机性，不让AI“自己说成功就成功”。 | **后端真随机**：在Node.js/Python后端使用`Math.random()`或随机库生成点数。<br>**流程**：AI思考时输出`{ "dice_check": {"skill":"侦查","difficulty":60} }`，后端执行掷骰，将结果`{ "dice_check":"侦查 成功", "roll":75 }`传回给AI，AI据此继续叙事。 |
| **Meta模式处理** | 理解并记录用户的叙事修改。 | 在系统提示词中定义“Meta指令”的优先级最高。用户切换模式时，发送特殊指令，要求AI只做“世界状态修改”，并返回修改摘要，不输出剧情。 |
| **存档与中断重连** | 保证随时退出，下次继续。 | 数据库全量保存：当前轮次、对话历史、物品列表、角色卡状态、世界摘要。打开页面时，直接从DB恢复所有状态，并将摘要和最近历史一起塞给AI生成开场回顾。 |

#### 🖥️ UI搭建要点

你的UI设计目标是**沉浸、低干扰、信息明确**。建议结构如下：

-   **左侧边栏（可收起）**：
    -   **角色卡信息区**：显示角色名、属性、技能，可点击查看详情。
    -   **物品栏区**：独立卡片式展示当前持有物品，新获得物品时有高亮动画。
    -   **故事控制**：存档/读档、退出按钮。
-   **中央主叙事区**：
    -   **流式叙事气泡**：AI的旁白/对话使用不同的气泡样式，支持Markdown。打字机效果输出，增加沉浸感。
    -   **Meta模式指示器**：当处于Meta模式时，整个界面边框微微变色，中央显示“故事暂停中”水印，清晰区分状态。
    -   **选项输入区**：普通模式下，叙事下方固定显示3个蓝色选项按钮，以及一个“自由行动...”输入框。点击按钮自动填入输入框并发送，或用户自行输入后按回车。
-   **顶部状态条**：
    -   显示当前章节/任务目标、Meta模式开关按钮。

**UI原型工具建议**：动手写代码前，先用Figma或甚至画一张草稿，确定布局。

### 3. 玩法的删改与拓展

#### ✂️ 可以为了速度先砍掉的功能
-   **多Agent团队模式**：这是研发黑洞，会让你的MVP开发周期延长3倍以上。**坚决砍掉**，聚焦“一个AI主持人 + 一个真人玩家”的单人极致体验。
-   **复杂的角色卡生成与调整**：MVP阶段，提供3个预生成模板，玩家只能选、改名字，属性固定。复杂的车卡系统以后再做。
-   **番外与完整复盘系统**：MVP只需在结局时生成一段总结性文字。精美的复盘海报、成就系统等，都是锦上添花。
-   **自由创建世界观**：**只提供2-3个你精调过的、剧情紧凑的短剧本**。这能让AI的表现可控到极致，体验远胜于开放式的漫无目的。

#### 🧩 未来可以拓展的方向（按优先级排序）
1.  **更深的记忆：从摘要升级到RAG**。当玩家抱怨“AI记不住40轮前我救过的NPC”时，引入向量数据库，实现精确检索记忆。
2.  **多Agent协作（简化版）**：不让多个Agent自主推演，而是让AI主持人在剧情中扮演多个不同性格的NPC。系统提示词里增加“角色切换”规则即可，远比多人模式简单，但能大幅提升“团队感”。
3.  **视觉化增强**：集成AI绘画API（如DALL·E 3/Stable Diffusion），在进入新场景、发现关键物品时，自动生成一张插图。这是提升沉浸感的利器。
4.  **异步多人**：等你单人体验打磨到极致，再考虑你最初设想的、真正的“多人异步跑团”。那时你会对AI工程化有更深的理解，去挑战那个真正的无人区。
