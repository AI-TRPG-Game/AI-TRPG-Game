## 世界设定阶段

**通用 API 参数设定：**

- `response_format: { "type": "json_object" }` —— 保证 API 返回合法 JSON 语法（DeepSeek 的 `json_object` 模式不支持传入自定义 schema，具体格式需通过 system prompt 传达）
- `temperature: 0.7` 

***

### 1. 进入世界设定

用户点击主页的 `<开启故事设定>` 按钮。

\[系统输出：世界设定阶段。]

\[系统在前端对话区输出一条固定指引，无需调用 LLM]：

> "你可以创造性地输入你想要的世界观雏形，它们可以是关键词描述，一句名人名言，一部电影，一段对话，或者一首诗！比如：“科幻，超级智能；信仰长久以来总能带给我们所有问题的解答。不过它现在却随着弗洛伊德和达尔文一起被冲进下水道；游戏《底特律》”。
>
> 然后我会为你扩写成完整的世界观与背景。调整好世界观后请点击'保存世界设定'按钮。"

此时对话区仅显示上述指引，等待用户输入。

***

### 2. 初次生成世界观

用户输入关键词描述，例如："蒸汽朋克与古老魔法并存，科技与信仰冲突剧烈的大陆"，并按下发送。

\[系统组装 LLM 输入]

#### 输出格式（世界设定）

LLM 必须输出一个 JSON 对象，格式如下（简称 `{WORLD_SCHEMA}`）：

```json
{
  "world_impression": "世界观印象（800-1000字）",
}
```

此 JSON Schema 通过 `system` prompt 文本直接告知 LLM。

#### LLM 输入 Schema

| roles 参数                 | 内容                                      |
| :----------------------- | :-------------------------------------- |
| `messages[0]` (`system`) | 系统辅助性指令 + 完整 `{WORLD_SCHEMA}` JSON 格式文本 |
| `messages[1]` (`user`)   | 用户当前 prompt                             |

> **Note – API 实际请求的 messages 示例：**
>
> ```json
> [
>   {
>     "role": "system",
>     "content": "我在尝试一种新型的AI跑团，旨在通过结合AI创作与跑团游戏元素，创造出文学性与娱乐性并重的RPG体验。你既是CoC7th规则下的KP，又是文学剧本创作者。\n现在我们先创建世界，请根据用户输入的文段生成世界观印象。为了营造代入感与沉浸感，你可以创造性地尝试环境切入/普通人视角/传说歌谣/对话切入/电影蒙太奇等手法的有机结合。\n\n你的输出必须是一个合法的 JSON 对象。请严格按照以下 JSON 格式输出，不要输出任何 JSON 以外的内容：\n\n{...}"
>   },
>   {
>     "role": "user",
>     "content": "..."
>   }
> ]
> ```

\[系统调用 LLM API，stream 方式将生成内容展示在对话区]

\[系统解析 LLM 返回的 JSON，检测 `world_description` 字段是否存在]

***

### 3. 调整与迭代

用户阅读后，可以在对话框中继续提出修改意见，如："魔法系统太弱了，我希望魔法是主流，科技反而被压制。"

\[系统重新组装输入，此次输入中包含之前的对话历史]

#### LLM 输入 Schema（调整轮次）

| roles 参数                    | 内容                                           |
| :-------------------------- | :------------------------------------------- |
| `messages[0]` (`system`)    | 系统辅助性指令 + 完整 `{WORLD_SCHEMA}` JSON 格式文本（同初次） |
| `messages[1]` (`user`)      | 用户首次 prompt                                  |
| `messages[2]` (`assistant`) | LLM 上次 JSON 输出原文                             |
| `messages[3]` (`user`)      | 用户最新修改意见                                     |
| ...                         | 可继续追加多轮                                      |

> **Note – 两轮调整后的 messages 示例：**
>
> ```json
> [
>   {
>     "role": "system",
>     "content": "……（系统指令 + {WORLD_SCHEMA}，同上）……"
>   },
>   {
>     "role": "user",
>     "content": "蒸汽朋克与古老魔法并存，科技与信仰冲突剧烈的大陆"
>   },
>   {
>     "role": "assistant",
>     "content": "{\"world_description\": \"在这片被称为'双律大陆'的土地上...\"}"
>   },
>   {
>     "role": "user",
>     "content": "魔法系统太弱了，我希望魔法是主流，科技反而被压制。"
>   }
> ]
> ```

\[LLM 生成新版本世界观 JSON 并展示]

用户可多次重复此步骤，直至满意。

***

### 4. 存档世界观

用户对当前对话中最后一条世界观描述满意后，点击对话区下方的 `<存档当前世界观>` 按钮。

\[系统解析最新 assistant 消息中的 JSON，提取 `world_description` 字段值]

\[系统将提取的文本填入右侧 `<世界观与背景>` 编辑模块，已有则覆盖，并自动存入 `world_settings` 字段]

\[系统在对话区提示："世界观已保存，你可以在侧边栏随时查看。如果不需要再调整，请点击`<开启人物设定>`按钮，进入下一阶段。"]

\[同时，`Game Orchestrator` 会将该世界观文本标记为当前会话的活跃设定。在后续对话中，重新输出给LLM，使其长期记忆该设定。]

***

### 5. JSON 输出保底机制

每次 LLM 输出后，系统尝试 `JSON.parse()` 解析。

- **若解析失败或缺少** **`world_description`** **字段**：自动重试一次，在 `system` 消息末尾追加 `【请一定只输出合法的 JSON，且必须包含 "world_description" 字段】`。
- **若重试后仍失败**：使用 LLM 完整输出原文作为备用，存入对话历史；系统提示用户"LLM 两次均未输出合法 JSON，已使用完整输出作为备用。"

***

### 前端设计总结

`<开启故事设定>` 按钮，`<存档当前世界观>`按钮，`<世界观与背景>`存储模块。

### 数据库设计总结

`world_settings` 字段：存储世界观与背景（纯文本，从 JSON 的 `world_description` 字段提取）
